This package contains the RMI and RMI-SSL link protocols. The RMI link protocol is used by default for inter-node communication and are mature and highly tested. This page is under construction. We hope to simplify the configuration of RMI advanced features

Configuration

To add features beyond the basic LAN and WAN templates, MTS Aspects must be added to the configuration file for each node.

Configuring SSL

SSL can be used to encrypt the RMI connections.

Enable SSL socket factory 

 -Dorg.cougaar.message.transport.server_socket_class=org.cougaar.mts.rmi.ManagedServerSocket

Add SSL link protocol 

 <component class="org.cougaar.mts.rmi.SSLRMILinkProtocol"
    insertionpoint="Node.AgentManager.Agent.MessageTransport.Component">
    
 <component class="org.cougaar.mts.base.LoopbackLinkProtocol"
    insertionpoint="Node.AgentManager.Agent.MessageTransport.Component">

Configuring Fail-over

RMI is not appropriate for some WAN and Firewalled environments. In these environments, you may want use RMI for inter-node communications among Nodes on the same LAN, but a different link protocol to access remote Nodes.

To enable the RMI link protocol all the following XML to all Nodes in a Runtime Configuration <component class='org.cougaar.mts.rmi.EnableRmiSenderSideAspect' insertionpoint="Node.AgentManager.Agent.MessageTransport.Component"> <component class='org.cougaar.mts.rmi.RMISocketControlAspect' insertionpoint="Node.AgentManager.Agent.MessageTransport.Component"> <component class='org.cougaar.mts.rmi.RMITestAspect' insertionpoint="Node.AgentManager.Agent.MessageTransport.Component"> <component class="org.cougaar.mts.rmi.RMILinkProtocol" insertionpoint="Node.AgentManager.Agent.MessageTransport.Component"> <component class="org.cougaar.mts.base.LoopbackLinkProtocol" insertionpoint="Node.AgentManager.Agent.MessageTransport.Component"> <component class="org.cougaar.mts.rmi.RmiAccessControlPlugin">
